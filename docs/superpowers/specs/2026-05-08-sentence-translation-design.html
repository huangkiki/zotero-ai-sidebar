Attention Is All You Need

Vaswani et al. — NeurIPS 2017

Abstract

The dominant sequence transduction models are based on complex recurrent or convolutional neural networks that include an encoder and a decoder. The best performing models also connect the encoder and decoder through an attention mechanism. We propose a new simple network architecture, the Transformer, based solely on attention mechanisms, dispensing with recurrence and convolutions entirely. Experiments on two machine translation tasks show these models to be superior in quality while being more parallelizable and requiring significantly less time to train.

1. Introduction

Recurrent neural networks, long short-term memory and gated recurrent neural networks in particular, have been firmly established as state of the art approaches in sequence modeling and transduction problems such as language modeling and machine translation. Recurrent models typically factor computation along the symbol positions of the input and output sequences. This inherently sequential nature precludes parallelization within training examples, which becomes critical at longer sequence lengths.

// 三档上下文：用 <target> 标记当前句，让模型只翻它，但能看见周边消歧
function buildPrompt(sentence, level, pageBundle, pageIdx) {
  if (level === 'sentence') return sentence;                  // 仅本句 · 最省

  const para = paragraphAround(pageBundle, sentence);   // 切到 \n\n 边界
  const marked = para.replace(sentence, `<target>${sentence}</target>`);

  if (level === 'paragraph') return marked;                  // 默认：本段 + tag

  const heading = nearestHeadingAbove(pageBundle, para);  // 用 font-size 启发
  return heading ? `## ${heading}\n\n${marked}` : marked;       // 最足：含章节
}

// 系统 prompt 固定（见第 08 节"系统提示词"）。模型只翻 <target> 内的内容。

从点击坐标到字符 offset

// src/context/sentence-detect.ts (new)
export function detectSentenceFromPoint(
  iframeDoc: Document,
  x: number, y: number,
  pageBundle: PageBundle,                      // from pdf-locator
): null | { text: string; rects: PdfRect[]; pageIndex: number } {
  const caret = iframeDoc.caretPositionFromPoint(x, y);
  if (!caret) return null;

  const pageOffset = walkTextLayerToOffset(caret);     // span-by-span concat
  const normOffset = pageBundle.rawToNormalized(pageOffset);  // pdf-locator helper
  const hit = findSentenceAt(pageBundle.normalizedText, normOffset);
  if (!hit) return null;

  const located = await locator.locate(hit.sentence);             // pdf-locator
  return { text: hit.sentence, rects: located.rects, pageIndex: pageBundle.pageIndex };
}

05浮层状态机

浮层在用户做完一次点击后会经过这五个状态。所有状态在交互效果图里都能切到。
idle
模式 OFF 或没点过；没有任何 overlay。
detecting
caret + splitter 工作中。通常 < 30ms，不显示中间状态以免闪烁。
streaming
浮层已挂出，原文已渲染，译文正在按 token 增量。光标闪烁。
done
译文完成；显示耗时。可保存 / 解释 / 下一句。命中缓存直接进此态。
error
provider 出错；浮层变红，原文保留，提供"重试"按钮。

06模块拆分

绿色 = 全新文件；黄色 = 现有文件需小改。

translate-mode 新

Toggle 状态、键盘快捷键 alt+T、cursor 切换、iframe 监听器 install/uninstall。每个 Reader tab 一份实例，跟 reader unload 一起释放。

sentence-detect 新

坐标 → caret → pageOffset → 调用 splitter → locate rects。复用 pdf-locator 的页面 bundle 缓存。

sentence-splitter 新

移植 zotero-pdf-translate 的算法。纯函数，独立测试。约 80 行 + abbr 表。

translator 新

仅 OpenAI。单次 text→text 流式调用 OpenAI Responses API，绕开 chat 的 tool loop 和 context-ledger。API 配置直接复用 chat 的 preset 列表（ModelPreset.apiKey/baseUrl），运行时按 provider==='openai' 过滤。但 model 独立：从该 preset 的 models[] 里挑一个，不强制和 chat 当前选的同款。复用 src/providers/openai.ts 的 fetch/auth 路径，参数独立：preset/model、reasoning.effort（off→省略字段；low/med/high → 直接传）、context window（sentence/paragraph/section → 决定 prompt 拼装时含多少周边文本，模型只翻 <target> 内）。

sentence-cache 新

键 = attachID + sha1(normalizedSent + srcLang + tgtLang + model).slice(0,16)。本地 prefs 存储 + sync snapshot 一段。带 LRU 上限。

overlay-dom 新

在 iframe document 里挂的浮层 DOM。位置由 sentence rects + viewport transform 推导，跟着页面缩放/滚动走或自动消失。

sidebar.ts 改

新增 toggle 按钮（Reader 工具栏旁）；mode 状态在 WindowSidebarState 加一字段；mount/unmount 时 install/uninstall 事件。

sync/state.ts 改

snapshot schema 加 sentenceTranslations section；apply 时 merge 而非 replace（避免把另一台机器新译的吞掉）。

preferences.xhtml + types.ts 改

新增三个 pref：targetLanguage / fastModelPreset / cacheMaxEntriesPerAttachment。

08默认值与预设

目标语言
固定简体中文，不出 UI、不可改（当前阶段不做多语言）；prompt 里写死 "...translate to Simplified Chinese..."
源语言
auto-detect（句子前 100 字符按 latin/CJK 比例判，主要为了 prompt 里说"from {src}"，识别错也不影响输出）
翻译 provider 范围
当前阶段仅适配 OpenAI。translator 内部不处理 Anthropic 路径——chat 那边可以照常用 Claude，但翻译走 src/providers/openai.ts 的 Responses API。后续扩 Anthropic 时，translator 抽出 provider 接口即可，不影响其他模块。
API 配置（preset） ⚙ 弹窗可调
直接复用 chat 的 preset 列表（同一份 storage.ts:loadPresets()）。下拉里 provider==='openai' 的 preset 可选，Anthropic preset 显示但 disabled。
默认值：第一个 OpenAI preset 的 id；如果用户一个 OpenAI preset 都没有，弹 inline 提示并打开 prefs。
pref 字段 translatePresetId: string
翻译模型 ⚙ 弹窗可调
独立于 chat 当前选的 model——从所选 preset 的 models[] 数组里挑（即 chat 里"添加候选模型"加进去的那些）。
默认值：所选 preset 的 model（即 chat 当前活跃的那个），用户可改。pref 字段 translateModel: string。
切换 preset 时这个字段自动复位到新 preset 的 model。
Thinking 程度 ⚙ 弹窗可调
默认 off（一般翻译不需要 reasoning，节省延迟和 token）。可选 low / medium / high，映射到 OpenAI Responses 的 reasoning.effort。pref 字段 translateThinkingLevel: 'off'|'low'|'medium'|'high'
上下文范围 ⚙ 弹窗可调
默认 paragraph（含本句所在自然段，让代词/术语能消歧）。可选 sentence（仅本句、最省 token）/ section（本节标题 + 整段）。
实现：splitter 已经把页面切成句子序列，"段落"= 当前句相邻无 \\n\\n 的连续句；"章节"= 向前回溯到最近的 heading（用 pdf-locator 的 heading 启发：font-size 突变 / 编号格式）。
构造 prompt 时只把上下文给 LLM 看，要求仅翻译目标句，prompt 里用 <target>...</target> 标记当前句。
pref 字段 translateContextLevel: 'sentence'|'paragraph'|'section'
固定快捷键
切模式 alt+T（Mac opt+T）；esc 关闭浮层；S/E 保存/解释。这几个写死，不开放自定义。
翻句快捷键 ⚙ 弹窗可调
设置弹窗里两个 key-recorder 录入：上一句（默认 Shift+Enter）/ 下一句（默认 Enter）。点击 recorder 进入录制态，下一次 keydown 即被记录；Esc 取消。
校验规则：必须含 Cmd/Ctrl/Alt/Shift 任一修饰键，或为 Enter/Tab/方向键这种不会和正文输入混淆的"裸键"；纯字母不允许（防止误绑住 chat composer 里打字）。
导航语义：走 splitter 给出的句子序列——同段内向前/后相邻；段末跨到下段首；首/末循环。
pref 字段 translatePrevSentenceShortcut、translateNextSentenceShortcut，存储格式是序列化的组合键字符串如 "Shift+Enter"、"Cmd+ArrowDown"。
浮层位置 ⚙ 弹窗可调
设置弹窗"浮层位置"行：句子上方（默认）/ 句子下方。一次设定全局生效，不再每次翻转。所选方向无空间时自动让位到另一侧，箭头朝向自动反转。pref 字段 translateOverlayPosition: 'above'|'below'
缓存上限
每 attachment 最多 2000 条（LRU）；超过则按 ts 淘汰最旧。注意：缓存 key 已经包含 model、thinking、context 三者，所以换设置不会污染之前的缓存。
"保存为注释" 行为
highlight color 用 #FFD400（Zotero 原生黄）；comment 写 [译] {translation}
系统提示词
"Translate to Simplified Chinese the sentence inside <target> tags. Use the surrounding context only to disambiguate pronouns and technical terms. Output only the translation, no explanation. Preserve technical terms in their original form when commonly used (e.g., Transformer, BLEU)."

10风险与缓解

caretPositionFromPoint 在 PDF.js textLayer 上不一定可靠
不同 PDF.js 版本对 textLayer 的渲染（绝对定位 div 还是 span）会变。验证矩阵：Zotero 7 / 8 / 9 默认 viewer。缓解：失败时回落到 elementFromPoint 找最近的 textLayer 节点，再用其 bounding rect 反查 anchor。
浮层在缩放/滚动时跟随困难
PDF.js 缩放是改 transform，textLayer 跟着走。缓解：浮层挂到 PDF.js 当前 page 的 textLayer DOM 里（同一个 transform 容器），让它跟着自然缩放；超出 viewport 时 IntersectionObserver 自动淡出。
浮层挡住下一句
线性阅读时用户的视线方向是向下，浮层若放在被点句下方会挡住即将要读的内容。缓解：默认锚点改在句子上方（覆盖已读内容），并删掉浮层里重复的英文原文（PDF 高亮已能定位）。仅当上方空间不足才翻到下方。结果：浮层高度从 ~180px 压到 ~95px，且默认覆盖的是已读区域。
PDF 多列布局 + 跨页句子
第二列点击映射到错列、句子被分页打断。缓解：splitter 工作在 page-level 文本而非 column-level；如最后一句不闭合（无终结符），向下一页 prefetch ±500 字续切。
缓存膨胀 / sync 文件膨胀
长论文翻完 1000+ 句 × 多 PDF 会推高 state.json 体积。缓解：每 attachment LRU 2000 上限；sync 时按 attachment 子表分块，pull 后惰性合并；提供"清空当前 PDF 缓存"按钮。
用户没有任何 OpenAI preset 时怎么办
翻译复用 chat 的 preset 列表，但只接受 provider==='openai' 的 preset。如果用户从来没配过 OpenAI preset（一直用 Claude chat），首次开模式会找不到可用 preset。缓解：(1) 翻译模式开启时先校验 presets.some(p => p.provider==='openai' && p.apiKey)，缺时浮层显示 inline 提示 "翻译需要 OpenAI preset · 去配置" 并附打开 prefs 的链接；(2) 设置弹窗"翻译 API"下拉里 Anthropic preset 用 disabled + 灰显，标注"暂不支持"；(3) 翻译失败显示具体错（401/429/网络/解析），不静默假装在转。
误把模式当作"chat 自动翻译"
该特性是显式 UI 动作，不在 chat 走 intent 路由。守则：translator 模块 import 不进 chat tool registry；agent-tools.ts 不暴露 translate 工具；prompt 里也不提"用户可能想翻译"。
数据流颜色约定：蓝 = 主路径，绿 = cache 短路，灰虚 = 显式用户操作支路。
所有 mockup 状态可在第 02 节顶部按钮切换；点 PDF 里别的句子浮层会跟着跳。
